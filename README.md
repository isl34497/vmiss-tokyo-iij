## 三、三网路由实测：去程 IIJ，回程直连

光看配置没用，日本 VPS 最关键的就是路由。VMISS 东京 IIJ 的官方说法是"电信 IIJ / 联通 IIJ / 移动 CMI"，但实际跑起来怎么样？综合多家第三方实测数据，情况大致是这样：

**电信方向**：去程从国内出口先经 NTT/Cogent 到东京，再对接 IIJ 骨干；回程从东京 IIJ 直连电信 163 骨干回国，广州 / 上海 / 北京三地延迟大致在 55–85ms 区间。晚高峰会有一定波动，但白天稳定性不错。

**联通方向**：去程走 CU2 9929 对接 NTT 到日本；回程走 IIJ 直连联通 4837 骨干，延迟普遍 80–110ms。实测里联通方向偶尔会绕一下 Softbank，但主路径仍是 IIJ 直连。

**移动方向**：去程先到香港 CMI 再上日本；回程 CMI 直连回国，广州 / 上海延迟约 55–90ms，是三条里表现最稳的一条。移动用户用这套线路做日常访问基本无压力。
